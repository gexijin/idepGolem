Data Format

iDEP works with four main file types. Preparing your files in these formats ensures a smooth upload and accurate downstream analysis.

  1. Gene expression matrix ( example1 , example2 )
  2. Fold-change and P-value table ( example )
  3. Experimental design file ( example )
  4. Gene-set file in GMT format ( example )

Quick demos are available through the Load demo button if you want to explore the interface before preparing your own files.

1. Gene Expression Matrix

Upload a CSV or tab-delimited text file in which each row represents a gene and each column represents a sample. The first column should contain gene IDs, and the first row should list sample names. A file size up to 100 MB is supported.

For RNA-seq, raw gene-level read counts are recommended so that iDEP can apply DESeq2 or limma-voom directly. Normalized matrices such as FPKM, RPKM, TPM, or data from microarrays and proteomics platforms are also accepted; you do not need to log-transform them before upload.

Gene IDs

iDEP converts a wide variety of identifiers to Ensembl IDs so that enrichment and pathway analyses are in sync. The system also uses IDs to guess the organism. Click the Info button beside the species selector to review coverage. If a gene cannot be matched it remains in the data under the original name, and analyses that do not require ID conversion (for example, clustering) still work.

When multiple identifiers map to the same Ensembl gene, iDEP keeps only the entry with the largest standard deviation to avoid duplicates. You can skip ID conversion entirely by checking Do not convert gene IDs to Ensembl on the Pre-Process page.

Sample Names

Column names define sample groups, so consistent naming is important. Use underscores to separate the group name from the replicate numberâ€”for example, Control_1, Control_2, TreatmentA_1, TreatmentA_2, TreatmentB_1, TreatmentB_2. Avoid hyphens or dots because they interfere with parsing. If your design is more complex, upload a separate design file as described below.

2. Experimental Design File

Upload an optional CSV or tab-delimited file with the same column names as the expression matrix. Each row provides additional sample annotations (factors) that iDEP can use to control confounders, perform paired tests, or model interactions.

A. Paired samples

Paired designs capture one-to-one relationships (for example, measurements taken before and after treatment on the same subject). Add a factor such as pairs alongside the treatment label, then select it as a block factor in the Stats tab so DESeq2 or limma can fit a model like ~ group + pairs.

D. 2Ã—3 and 3Ã—3 designs

Two genotypes combined with three treatment conditions constitute a 2Ã—3 design. Similarly, three cell lines with two treatments form a 3Ã—3 layout. Maintain one column per sample and provide two factors describing genotype and condition.

E. Three or more factors

Complex clinical datasets sometimes include many factors (sex, subtype, batch, etc.). Be mindful that statistical models require enough samples to estimate each term: dozens of factors with only a few samples will fail. iDEP supports straightforward models; interaction terms in limma are currently limited to two-factor designs.

Sample Size Guidelines

Differential-expression tools such as limma and DESeq2 require at least two biological groups with two or more replicates each. Exploratory plots (PCA, clustering, k-means) can operate with as few as two columns.
